Le programme ne prend qu'une image instantané = pas une mise à jour des évolutions

Veiller à enregistrer votre code dans un repository GitHub & à effectuer des commis réguliers et clair.


Ordre d'action suggérer :

1. Choisir n'importe quel page de BookstoScrape, écrire un scrip qui visite le site 
et en extrait d'un produit les informations suivantes:

product_page_url
universal_ product_code (upc)
title
price_including_tax
price_excluding_tax
number_available
product_description
category
review_rating
image_url

2. Ecrivez les données dans un fichier CSV -> utilise les champs ci-dessus comme colonnes

3. Maintenant, faire la même chose mais qui extrait tout les produit d'une seul catégories
3.1 Attention ! Certaines pages comprennent plus que 20 produits et donc plusieurs pages ! (Pagination:::)

4. Ensuite, étendre le travail pour que le script traverse tout le site : 1 catégories = 1 fichier CSV

5. Finalement, prolonger le script pour qu'il télécharge et enregistre le fichier image de chaque page produit que vous consultez


(Dans le GitHub !) -> Enregistrer un fichier "requirements.txt" sans enregistrer son environnement virtuel. 
(Dans le GitHub !) -> Ne pas non plus y enregistrer mes propres fichiers .CSV
(Dans le GitHub !) -> Prendre le temps d'écrire un "README.md", à ajouter au repository


	LIVRAVBLES 
- Un lien vers le github qui doit contenir :
	1. L'ensemble du code d'application
	2. Le fichier requirements.txt, mais PAS l'environnement virtuel (.gitignore + /env)
	3. Un fichier README.md expliquant comment créer et activer l'environnement virtuel, puis exécuter le code d'application
	4. Les données/images extraites NE DOIVENT PAS FAIRE PARTIE DU REPOSITORY LUI-MÊME

- Un fichier compréssé contenant toutes les données extraites et les images associées dans nu format ou une structure facile à suivre.
